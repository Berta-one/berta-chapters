# Chapter 25: AI Governance & Ethics

**Track**: Advanced | **Time**: 6 hours | **Prerequisites**: Chapters 1, 20, 22

---

Capability without **governance** is a liability. This final chapter turns responsible-AI principles into operational tools: a rule-based **risk classifier** in the spirit of the **EU AI Act**, a **model card** generator and validator for documentation and accountability, and a **risk matrix** for likelihood-times-severity assessment. It closes the curriculum by connecting everything you've built to the frameworks — legal, organizational, and ethical — that decide whether it ships responsibly.

---

## Learning Objectives

By the end of this chapter, you will be able to:

1. **Classify systems by risk** with an EU AI Act-style rule set (minimal → unacceptable)
2. **Generate model cards** that document intended use, data, metrics, and limitations
3. **Validate documentation** completeness before release
4. **Assess risk** with a likelihood × severity matrix
5. **Apply fairness and transparency** principles to real decisions
6. **Navigate regulation** — the EU AI Act, NIST AI RMF, and data-protection basics
7. **Define accountability** — owners, audits, and incident response
8. **Build an ethical culture** — review boards, escalation, and long-term strategy

## What You'll Build

- **Risk classifier** — map a system's use-case flags to a regulatory risk tier
- **Model card generator** — structured metadata into a Markdown model card
- **Card validator** — flag missing required documentation fields
- **Risk matrix** — likelihood × severity into low/medium/high/critical
- **Governance checklist** — the gates a system clears before deployment
