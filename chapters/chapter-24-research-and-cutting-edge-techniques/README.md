# Chapter 24: Research & Cutting-Edge Techniques

**Track**: Advanced | **Time**: 8 hours | **Prerequisites**: Chapters 1, 9, 11, 18

---

Staying at the frontier means **reading papers and implementing them**. This chapter builds that muscle: you reproduce two foundational components from scratch — **scaled dot-product attention** (the core of every transformer) and a **mixture-of-experts** top-k router (the scaling trick behind today's largest models) — in pure NumPy. Along the way you learn how to read a paper efficiently, turn equations into code, and sanity-check an implementation against the properties the math guarantees.

---

## Learning Objectives

By the end of this chapter, you will be able to:

1. **Read a paper efficiently** — abstract, figures, equations, then details
2. **Translate equations to code** and verify against mathematical invariants
3. **Implement scaled dot-product attention** and confirm its softmax properties
4. **Implement mixture-of-experts** top-k gating and sparse expert combination
5. **Reason about scaling** — why MoE buys capacity without proportional compute
6. **Sanity-check implementations** with shape and probability assertions
7. **Survey current frontiers** — long context, MoE, multimodality, reasoning
8. **Plan a reproduction** of a paper end to end

## What You'll Build

- **Scaled dot-product attention** — the transformer's core operation in NumPy
- **Softmax with invariants** — numerically stable, rows sum to one
- **Top-k MoE router** — gate, select experts, renormalize weights
- **Sparse MoE forward** — combine only the selected experts per token
- **Verification harness** — assert the properties the paper's math promises

## Quick Start

2. **Create a virtual environment and install dependencies**
   ```bash
   python -m venv .venv

   ```

