# Chapter 20: Building Production AI Systems

**Track**: Advanced | **Time**: 10 hours | **Prerequisites**: Chapters 1, 2, 3, 4, 5, 6, 15

---

Production AI is mostly **systems engineering** wearing an ML hat. Real traffic brings rate limits, partial failures, latency spikes, and the need to ship changes safely. This chapter builds the reliability toolkit: a **token-bucket rate limiter**, a **circuit breaker** for failing dependencies, **exponential backoff**, and **canary routing** for safe rollouts — then frames observability, cost, and the organizational realities of running AI in production. Everything is pure-stdlib and testable.

---

## Learning Objectives

By the end of this chapter, you will be able to:

1. **Rate-limit** requests with a token-bucket and reason about burst vs sustained load
2. **Contain failures** with a circuit breaker (closed / open / half-open)
3. **Retry safely** with exponential backoff and jitter
4. **Roll out safely** with deterministic canary routing and rollback
5. **Instrument** services with structured logs and the golden signals
6. **Reason about SLOs** — latency percentiles, error budgets, and availability
7. **Control cost** — caching, batching, and right-sizing inference
8. **Navigate** the human side: on-call, incident review, and technical debt

## What You'll Build

- **Token-bucket limiter** — burst-tolerant rate limiting with refill over time
- **Circuit breaker** — trips open after repeated failures, recovers via half-open
- **Backoff schedule** — exponential delays with optional jitter
- **Canary router** — deterministic hash-based traffic splitting for safe rollouts
- **Structured logger** — JSON events carrying the golden signals

## Quick Start

1. **Enter the chapter**
   ```bash
   cd chapters/chapter-20-building-production-ai-systems
   ```

2. **Create a virtual environment and install dependencies**
   ```bash
   python -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **Run the notebooks in order**
   ```bash
   jupyter notebook notebooks/
   ```
   Start with `01_reliability_patterns.ipynb`, then work through 02 and 03.
